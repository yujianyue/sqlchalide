代码简介

查立得PHP+mysql通用已有数据表查询系统19查搜模式合1 V2.0

推荐环境: 宝塔/护卫神/主流虚拟主机 PHP5.4-7.3/mysql5.6环境

一般网站站长/程序员/计算机熟练者自用,快速用于已有数据表的查询搜索。

单文件实现19查搜模式,每个模式均通用Mysql单表(灵活字段和查询条件)。

使用步骤

准备:已绑定域名的php网站空间(5.4-7.3)+Mysql5.6数据

第1步：上传代码

用你的方式保持原有结构将代码上传到新创建的英文字母组成的名称的文件夹(比如soulide)下；

上传方式：FTP,宝塔面板在线上传,在线文件管理面板,远程桌面等；

访问测试：访问绑定的域名+文件夹,参考格式：http://96448.cn/soulide/；

第2步：修改参数

在线编辑数据:(宝塔等在线工具)修改 index.php 参数;也可本地修改(别用记事本修改)后上传替换。

默认打开报错信息方便查找原因；自己可以根据报错信息百度到原因。正式使用可以关闭报错信息。

第3步：查询测试

查询测试第一条、最后一条数据；再抽查多条其他条数据。

其他特殊情况查询测试:比如大小写、首尾有空格的等。

经验技巧

使用要点：

1. 查询条件:推荐设置2-3个，唯一对应用户的字段(比如学号、工号、学籍号)+验证字段(名字、生日等)!

2. 必要原则:不上传非必要字段,比如查询成绩就无需上传身份证号信息,如果是查询条件可以设置隐藏列!

3. 数据意识:(参考access,mysql等二维结构表)Excel首行字段即列标题，以后一行一条数据。

4. 严谨:字段设置和数据库一致；设置的参数尽量不要有符号特别是回车换行单引号双引号等。

5. 性能优化:精准查询可以查询条件字段做索引;开启opcache缓存!

6. 尚未有数据表(自己有导入数据能力)：自己可以用自己的方式导入数据再使用本系统!

特殊字段处理

1. 隐藏列:搜索代码文件“隐藏列:”,字段存在则生效,多字段+隔开!

2. 拨号链接:单元格为tel:开头的手机号，则自动加拨号链接!

3. 图片链接:单元格img:开头的http/https开头的网址，则显示为图片!

4. 网址链接:http://或https://开头的单元格自动加链接!

5. 按值隐藏:表格习惯-或/、按数据库习惯Null值字段将隐藏!

6. 无合并单元格:请先拆分合并后填充数据,不然只有第一格有内容其他格为空！
